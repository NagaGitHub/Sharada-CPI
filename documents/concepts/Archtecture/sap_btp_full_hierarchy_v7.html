Architecture Key · Relationships
App → XSUAA (blue)At runtime, app exchanges client credentials (from VCAP_SERVICES) for a JWT token. Token authenticates outbound API calls.
App → Destination (green)At runtime, app calls Destination API to look up a named destination's URL, auth type, and proxy config.
App → Connectivity (orange-red)App sends on-premise requests through Connectivity proxy, which relays them via Cloud Connector tunnel.
IS → SAP Infra (amber)Integration Suite deploys iFlows to IS Runtime Nodes in SAP Global Infrastructure. iFlows then call XSUAA, Destination, Connectivity.
Subaccount PatternBest practice: 1 subaccount per environment — mycompany-dev, mycompany-prod. Each has isolated XSUAA tenant, quotas, and roles. Do NOT use spaces as dev/prod boundary.
Space Names (non-prod)dev · test · int · ci · uat · staging · preprod
Space Names (prod)preprod · prod · hotfix — in a dedicated prod subaccount
Service KeyCredentials for external apps (not in CF). Contains client_id, secret, URLs.
Service BindingCredentials injected into VCAP_SERVICES for CF apps in the same space.
CCDBSingle source of truth for all CF metadata. No compute — records only.
Diego CellsShared across ALL spaces. Scheduler places containers freely (no dedicated cell per space).
IS RuntimeNOT Diego Cells. K8s pods in SAP Global Infra. iFlows run here.
Cloud ConnectorOn-premise agent. Outbound tunnel to Connectivity Server. No inbound firewall rules needed.
