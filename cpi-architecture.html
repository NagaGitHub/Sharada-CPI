SAP CPI architecture primer

See how SAP CPI fits into BTP, deploys on Cloud Foundry, and executes iFlows at runtime.

This page turns SAP Community and SAP Help architecture notes into a step-by-step visual model. It focuses on the control plane, runtime nodes, integration artifacts, adapters, lifecycle, and the exact path a message follows from a sender system to a receiver system.

Control Plane Tenant Management Node handles discovery, design, administration, deployment coordination, and monitoring entry points.
Runtime Plane Worker or runtime nodes process messages, run deployed integration flows, and host the integration engine.
Tenant Model Customer tenants are isolated with separate resources and database schemas while sharing SAP-managed infrastructure.
iFlow Model Integration flows define senders, receivers, adapters, processing steps, resources, routing, and operational behavior.
